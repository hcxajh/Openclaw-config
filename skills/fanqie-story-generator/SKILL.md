# 番茄小说短故事生成

## 触发条件
用户说：生成短故事 / 创作故事 / 写小说

## 输入参数
| 参数 | 说明 | 示例 |
|------|------|------|
| 数量 | 要生成几个故事 | 10个 |
| 主题 | 标题或题材 | 婆媳关系、养老问题等 |
| 读者 | 目标年龄段 | 50-70岁中老年 |

## 文件保存路径
```
fanqie/YYYY-MM-DD/{编号}/
```

## 文件结构
```
fanqie/2026-03-13/
├── 001/
│   ├── 婆婆的拆迁款引发的家庭风波.txt
│   └── 婆婆的拆迁款引发的家庭风波_cover.png
├── 002/
│   ├── 儿媳怀孕后对我的态度转变.txt
│   └── 儿媳怀孕后对我的态度转变_cover.png
└── ...
```

## 执行流程（按顺序执行）

### 🚨 禁止并行执行！必须逐个生成

**警告**：禁止使用并行subagent或同时生成多个故事，否则会导致：
- 管理混乱
- 文件命名冲突
- 字数统计错误

### 每次只生成1个故事，循环执行

1. **创建文件夹**
   - 路径：`fanqie/YYYY-MM-DD/{编号}/`
   - 编号格式：001, 002, 003...
   - **先检查是否已存在**：如果已存在则跳过该编号

2. **生成故事正文**
   - 使用LLM生成故事内容
   - **每个故事8000-9000字（最低8000字）**
   - 在prompt中明确要求：8000-9000字
   - **不要有标题**（即文件开篇直接是正文）
   - **不要用##等章节标记**，用自然段落过渡
   - 结构：八步法（主人公→目标→障碍→行动→结果→转折→高潮→结局）
   - 风格：情节曲折、冲突强烈、有反转、正能量

3. **字数验证（关键步骤！）**
   - 生成后立即统计字符数：`wc -m {文件路径}`
   - 中文字数 ≈ 字符数 / 2
   - **如果低于16000字符（8000字），必须重新生成**
   - 直到达标为止

4. **保存TXT文件**
   - 路径：`{编号}/{标题}.txt`
   - 编码：UTF-8
   - **文件名不要带编号前缀**！直接用标题命名

4. **生成封面图片**
   - **使用 ComfyUI**（本地服务）
   - **如果 ComfyUI 不可达（连接超时/拒绝），跳过封面生成，继续下一个故事**
   - **严禁使用 Minimax 生成封面**
   - 命令：
   ```bash
   python ~/.openclaw/skills/comfyui/scripts/txt2img.py \
     --prompt "Chinese elderly couple, warm family scene, traditional Chinese house, soft lighting, realistic style, 9:16 vertical aspect ratio" \
     --width 576 \
     --height 1024 \
     --model flux1-dev.safetensors
   ```
   - 提示词要点：英文描述、包含"9:16 vertical"、体现故事主题

5. **移动封面文件**
   - ComfyUI输出到：`{当前工作空间}/comfyui/{日期}/`
   - 移动到：`{编号}/{标题}_cover.png`

6. **汇报进度**
   - 每生成完1个故事，汇报1次

## 输出格式
```
✅ 完成！现在共有N个故事

| 编号 | 标题 | 正文 | 封面 |
|------|------|------|------|
| 001 | 婆婆的拆迁款引发的家庭风波 | ✅ TXT | ✅ PNG |
| 002 | 儿媳怀孕后对我的态度转变 | ✅ TXT | ✅ PNG |

文件位置：`fanqie/2026-03-13/`
```

## 注意事项

1. **🚨 禁止并行执行**：禁止使用subagent并行生成故事！必须串行执行
2. **字数验证**：生成后立即验证，低于8000字必须重写
3. **网络超时处理**：API调用超时要重试
4. **敏感词处理**：如果封面因敏感词失败，更换提示词重试
5. **ComfyUI配置**：
   - 地址：`http://10.1.1.90:8188/`
   - 首次生成需加载模型，约30-40秒
   - 后续生成约10-15秒
6. **封面尺寸**：番茄小说封面推荐 9:16 竖版 (576x1024)
7. **文件名检查**：保存后检查文件名是否有编号前缀，如有则重命名
